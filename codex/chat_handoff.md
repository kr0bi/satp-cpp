## 2) Dataset Evolution and Current Format

### 2.1 Historical evolution (important context)

- Initial text datasets and C++ generation utilities were removed/deprecated.
- JSON dataset phase existed but was later dropped for size/performance reasons.
- Current standard is a **binary compressed format** read by C++ and generated by Python.

### 2.2 Current binary dataset format (v2)

Key points:

- File is partitioned; each partition is compressed independently (`zlib`).
- C++ loads **one partition at a time** in RAM.
- Includes truth information for streaming prefix analysis \(F_0(t)\).

Header:

- `MAGIC=SATPDBN2`
- `VERSION=2`
- global `n`, `d`, `p`, `seed`

Partition table entry:

- `ENTRY_SIZE = 60` bytes from struct layout `<QQQQQQIII>`
- 6 × `Q` (uint64) + 3 × `I` (uint32) = 48 + 12 = 60
- fields: offsets/sizes for values payload and truth payload, local `n,d`, encodings.

Encodings:

- values: `ENCODING_ZLIB_U32_LE`
- truth bits: `ENCODING_ZLIB_BITSET_LE`

Truth semantics:

- bit `b_t=1` iff element at position `t` is first occurrence in that partition
- exact prefix cardinality reconstructed as:
  \[
  F_0(t)=\sum_{i=1}^{t} b_i
  \]

### 2.3 Naming

Dataset naming convention in use:

- `dataset_n_{n}_d_{d}_p_{p}_s_{seed}.bin`

## 6) Type A / Type B Analysis Protocol (Advisor-Driven)

Advisor requested clearer graphs and interpretation.

### Type A

- x-axis: **real cardinality** on prefix, \(F_0(t)\)
- y-axis: estimated cardinality, \(\hat F_0(t)\)
- compare algorithms, include uncertainty (stddev/intervals)

### Type B

- x-axis: average repetitions on prefix:
  \[
  \rho_t = \frac{t}{F_0(t)}
  \]
- y-axis: \(\hat F_0\) at selected target cardinalities \(F_0^\*\)

### Prefix-constant-rho generation rationale

For controlled behavior:

- enforce `n % d == 0`
- \(\rho = n/d\)
- one new distinct every \(\rho\) items (block protocol)
- then \(F_0(j\rho)=j\)

This was introduced specifically to decouple effects and make Type A/Type B interpretability stronger.

### 16.4 Checkpoint planning note

The old checkpoint generator was too front-loaded and could exceed the requested budget on small cases.
